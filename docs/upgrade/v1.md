# Upgrade Guide v1

This page tracks the repository-side upgrade contract for the `1.x` platform line before the formal versioning ADR is fully implemented.

## Current guidance

1. Read the latest release notes and `Unreleased` changelog summary before applying `main`.
2. Run `make validate` from a clean checkout.
3. Regenerate generated surfaces with `make generate-status`.
4. Build the public documentation surfaces with `make docs`.
5. Use the promotion and live-apply workflow that matches the affected services.
6. Record any live apply with a receipt and update the relevant workstream state.

## Compatibility assumptions

- Repository releases remain the source of truth for automation changes.
- Platform version bumps happen only after a verified live apply from `main`.
- Public documentation and changelog publication should be rebuilt before edge publication changes are applied.

This page will expand into the formal upgrade path once ADR 0110 is implemented.
