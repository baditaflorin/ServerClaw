# Upgrade Guide: v1.0.0

## Scope

This guide covers upgrades from any `0.x` repository release to `1.0.0`.

## Summary

`1.0.0` is defined as the first product-grade LV3 platform milestone. It is a stabilisation release:

- no manual migration steps are required
- no service removals are allowed
- no network identity changes are allowed
- no secret path or startup format changes are allowed

## Operator Procedure

1. Read the release notes in `RELEASE.md` or `docs/release-notes/<version>.md`.
2. Run `lv3 release status` and confirm there are no active release blockers.
3. Confirm the repository is on the target tagged `main` release.
4. Apply the merged automation from `main` using the normal promotion and live-apply path.
5. Verify the resulting live platform receipts and, if a live change was applied, update `versions/stack.yaml` from `main`.

## Expected Outcomes

- repository version reaches `1.0.0`
- platform version remains unchanged until a deliberate live apply is completed from `main`
- no manual post-upgrade remediation is required

## Unsupported Paths

- skipping directly from `1.x` to `3.x`
- applying a future major release without reading its dedicated upgrade guide

Future major releases must add their own `docs/upgrade/v<major>.md` guide before they are cut.
