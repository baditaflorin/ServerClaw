# Release 0.171.0

- Date: 2026-03-26

## Summary

Add 6 new foundational ADRs that enable LLM agents to self-discover repository structure and capabilities with minimal token overhead. These ADRs establish machine-readable indexes, metadata standards, and context-preservation protocols for seamless multi-agent coordination.

## Key Features

### Repository Discovery Infrastructure (ADR 0163-0164)
- **ADR 0163**: Repository structure index - machine-readable directory and component registry for fast agent onboarding
- **ADR 0164**: ADR metadata index - canonical catalog of all decision records with decision status and implementation metrics

### Agent Operational Standards (ADR 0165-0166)
- **ADR 0165**: Playbook and role metadata standard - discoverable contracts for Ansible automation components
- **ADR 0166**: Canonical configuration locations - unified registry of where configuration data lives across the platform

### Multi-Agent Coordination (ADR 0167-0168)
- **ADR 0167**: Agent handoff and context preservation - protocols for passing work and state between agents without context loss
- **ADR 0168**: Automated enforcement of agent standards - validation gates that prevent agents from drifting from ADR-defined patterns

## Platform Impact

- Repository version advances to 0.171.0
- Agents can now discover repository structure with single token-efficient lookup (no manifest reading needed)
- Configuration and automation discovery time reduced from minutes to seconds
- Enables new workflows: cross-agent handoffs, context-aware parallelization, self-healing automation

## Design Principles

All 6 ADRs prioritize **token efficiency** for LLM agents:
- Machine-readable JSON indexes (not prose) for fast parsing
- Flat, discoverable structure (no deep nesting)
- Metadata includes enough to route between agents without reading entire files
- Validation gates prevent drift that would require expensive re-discovery
