# Release 0.120.2

- Date: 2026-03-24

## Summary
- Fixed the live Keycloak password-reset flow so `https://ops.example.com` and `https://sso.example.com` no longer fall into the generic identity-provider error page during reset requests.
- Moved Keycloak's repo-managed SMTP configuration to the VM-private Stalwart relay on `10.10.10.20:1587`, avoiding the public STARTTLS certificate trust failure that blocked recovery mail.
- Hardened both the mail-platform and Keycloak runtimes against the recurring Docker nat-chain drift on `docker-runtime`, including automatic Docker restart, readiness probing, and force-recreate behavior when published ports need to be repaired.
- Corrected the live Keycloak database target to the reachable PostgreSQL primary host instead of the currently broken HA VIP path.

## Platform Impact
- repository version advances to `0.120.2`; live platform version advances to `0.114.6` after the Keycloak reset-mail recovery was applied on `docker-runtime`

## Residual Risk
- Password-reset requests now render and submit successfully, but this still depends on controller-local recovery secrets under `.local/keycloak/` and `.local/mail-platform/`.
- The PostgreSQL HA VIP path for Keycloak remains unhealthy in the live environment, so Keycloak is intentionally pinned to the declared initial primary until that HA reachability issue is fixed separately.
