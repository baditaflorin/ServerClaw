# Release 0.98.0

- Date: 2026-03-23

- implemented ADR 0111 in repository automation with an environment-aware integration test runner at `scripts/integration_suite.py`, including structured JSON reporting and explicit `gate`, `nightly`, `all`, and `destructive` modes
- added the repo-managed live integration suite under `tests/integration/` for authentication, platform API, secret rotation, deployment, observability, data services, failover, and backup recovery coverage, with a default-safe network opt-in guard
- wired the non-destructive gate path into `config/validation-gate.json`, added `make integration-tests` and `make nightly-integration-tests`, and registered the nightly workflow in the workflow and command catalogs
- added the Windmill wrapper `config/windmill/scripts/nightly-integration-tests.py`, including optional Mattermost and GlitchTip notifications plus a `uv` fallback when `pytest` is not preinstalled on the worker
- documented the operator workflow, environment variables, report locations, and destructive toggles in `docs/runbooks/integration-test-suite.md`
- updated ADR 0111 plus the workstream registry to mark the repository implementation as complete in `0.98.0`

Platform impact:

- no live platform version bump; this release adds repository-side integration verification, reporting, and nightly workflow contracts only
