# Release 0.101.0

- Date: 2026-03-23

- completed ADR 0092 in repository automation by adding the FastAPI-based unified platform API gateway under `scripts/api_gateway/`, including JWKS-backed bearer-token validation, native `/v1/platform/*` endpoints, aggregate health handling, OpenAPI publication, and generic upstream proxy routing
- added `config/api-gateway-catalog.json` plus schema validation, a dedicated gateway runtime dependency set in `requirements/api-gateway.txt`, and regression coverage for both the catalog loader and the gateway application
- added the collection-native `api_gateway_runtime` role and service playbooks to build and deploy the gateway on `docker-runtime`, wire the optional legacy platform-context token bridge, and provision the repo-managed Keycloak `api-gateway` client secret mirror
- extended canonical platform topology, service capability, control-plane lane, API publication, subdomain, health-probe, and Uptime Kuma catalogs so `api.example.com` is modeled as the unified public API entrypoint and `api_gateway` is a first-class governed service
- updated the agent tool registry so platform-context queries resolve through `https://api.example.com/v1/platform-context/v1/context/query`, documented gateway operations in `docs/runbooks/configure-api-gateway.md`, and added a dedicated Grafana dashboard definition for gateway request health
- repaired the repo-wide validation baseline by making the shared OpenTofu `proxmox-vm` module expose a configurable `prevent_destroy` lifecycle switch and by updating the service-catalog regression to reflect the new `api_gateway` service entry

Platform impact:

- no live platform version bump; the gateway is implemented in repository state only and still requires apply from `main` before `api.example.com` is claimed live
