# Release 0.101.0

- Date: 2026-03-23

- implemented ADR 0098 in repository automation by adding the new `postgres-replica-lv3` guest definition, Patroni-managed PostgreSQL HA, a three-member etcd quorum, and keepalived-based VIP failover for the database tier
- moved the stable Postgres service endpoint to `database.lv3.org` on VIP `10.10.10.55` and updated Keycloak, Mattermost, NetBox, OpenBao, and Windmill to consume the HA endpoint instead of a single node IP
- added repo-managed failover visibility through updated health probes, Uptime Kuma monitors, Grafana dashboards, service capability metadata, and the `docs/runbooks/postgres-failover.md` operational procedure
- updated ADR 0098 plus the workstream registry to mark the repository implementation complete in `0.101.0` while keeping live rollout explicitly pending until applied from `main`

Platform impact:

- no live platform version bump; this release adds repository-side Postgres HA automation, topology, and runbooks only
