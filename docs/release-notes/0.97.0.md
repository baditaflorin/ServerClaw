# Release 0.97.0

- Date: 2026-03-23

- implemented ADR 0094 in repository automation with a generated MkDocs-based documentation site at `build/docs-portal/`, sourced from the service catalog, subdomain catalog, secret catalog, ADR corpus, runbooks, release notes, and a generated OpenAPI snapshot
- added the `make docs` and `make deploy-docs-portal` entry points, wired docs-site generation into `make generate-status` and generated-portal validation, and documented the portal workflow in `docs/runbooks/developer-portal.md`
- introduced the initial developer-facing runbooks for deploying a service, rotating certificates, adding a new service, and break-glass recovery so the published docs site has operator-oriented starter content alongside the existing runbook corpus
- registered `docs.lv3.org` as a governed public hostname through the service catalog, subdomain catalog, and shared edge publication defaults, including a temporary `X-Robots-Tag: noindex` header until the platform reaches `1.0.0`
- updated ADR 0094 plus the workstream registry to mark the repository implementation as complete in `0.97.0`
- implemented ADR 0106 in repository automation by moving the governed ephemeral VM pool to `910-979`, enforcing owner/purpose/expiry tags during fixture creation, and adding direct VMID-aware teardown support
- added the seed [capacity model](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/config/capacity-model.json), the [ephemeral VMID guard](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/scripts/validate_ephemeral_vmid.py), and validation coverage in [scripts/validate_repository_data_models.py](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/scripts/validate_repository_data_models.py)
- upgraded the Windmill reaper path to [config/windmill/scripts/ephemeral-vm-reaper.py](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/config/windmill/scripts/ephemeral-vm-reaper.py), kept the receipt history flow intact, and extended the static ops portal with an `Ephemeral VMs` panel
- expanded `lv3 fixture` routing with `create` and `destroy` lifecycle arguments, updated the ephemeral-fixtures and scaffold-new-service runbooks, and marked ADR 0106 plus its workstream as implemented in `0.97.0`
- completed ADR 0093 in repository automation by adding the interactive ops portal runtime under `scripts/ops_portal/`, with server-rendered HTMX/Jinja views for platform overview, service actions, drift, runbooks, changelog, and an SSE deployment console
- added the new `lv3.platform.ops_portal_runtime` role plus `playbooks/ops-portal.yml` and `playbooks/services/ops-portal.yml` so the portal can be built and converged on `docker-runtime-lv3` from repo-managed automation
- switched the `ops.lv3.org` edge contract from static-file publication toward protected proxy mode, including forwarded `X-Auth-Request-*` identity headers, forwarded access tokens from `oauth2-proxy`, and an unauthenticated `/health` endpoint for upstream probing
- registered the interactive portal contract across the service, health, subdomain, and API gateway catalogs, and added the matching Uptime Kuma monitor plus docker-runtime firewall allowance for port `8090`
- preserved the legacy static portal generator as a read-only fallback by archiving the generated landing page to `receipts/ops-portal-snapshot.html`
- updated ADR 0093 and its workstream metadata to mark the repository implementation complete in `0.97.0`, while keeping live cutover explicitly pending until applied from `main`

Platform impact:

- no live platform version bump; this release adds repository-side interactive operations, documentation publication, lifecycle enforcement, and validation only
