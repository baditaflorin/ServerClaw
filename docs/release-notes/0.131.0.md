# Release 0.131.0

- Date: 2026-03-25

## Summary
- implemented ADR 0126 with a durable observation-to-action closure loop that carries one finding through triage, proposal, execution, verification, approval, and explicit resolution states
- added repo-managed closure-loop persistence, transition ledger events, and `platform.incident.*` publication for incident milestones
- replaced the Windmill observation-loop placeholder with a real wrapper that turns findings into durable loop runs and terminates automatically when the goal is already met or verification succeeds
- added `lv3 loop start|status|approve|close` so operators can inspect, approve, and close paused runs directly from the CLI

## Platform Impact
- repository version advances to `0.131.0`
- platform version remains `0.130.1` until this repository release is applied live from `main`

## Residual Risk
- conflict detection is still service-scoped until ADR 0127 resource claims are widened into the closure loop
- verification still falls back to current service health surfaces when a workflow does not declare an explicit verification block
- resolved runs are recorded durably, but automatic case-library promotion remains a follow-up beyond ADR 0126
