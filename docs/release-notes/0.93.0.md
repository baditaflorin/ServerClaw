# Release 0.93.0

- Date: 2026-03-23

- completed ADR 0087 in repository automation by adding the fast-hook and remote-first pre-push validation gate, the shared `config/validation-gate.json` manifest, the hook install target, explicit bypass receipts, gate-status reporting, and the Windmill post-merge gate runner
- aligned the gate runner with the ADR 0084 Packer pipeline by validating through `scripts/validate_packer_templates.sh`, mounting the build-worker Packer plugin cache into the Docker check runner, and excluding transient local fallback artifacts from the Trivy scan
- documented the operator workflow for installing, running, bypassing, and troubleshooting the validation gate, including the build-worker Packer plugin cache prerequisite
- updated ADR 0087 and the workstream registry to mark the validation gate as implemented in repo version `0.93.0`
- fixed `playbooks/provision-subdomain.yml` so catalogued Hetzner subdomain converges load controller defaults correctly
- switched the shared NGINX edge certificate path from HTTP-01 webroot validation to a pinned `certbot-dns-hetzner` DNS-01 flow isolated in a dedicated plugin virtualenv
- added focused regression coverage for the subdomain provisioning playbook and the NGINX edge publication role
- kept the Hetzner DNS API token out of git by consuming it from controller-local state and writing the edge credential file only on the live host

Platform impact:

- no live platform version bump in this release commit; the live publication receipt and platform-state updates are recorded separately after the successful apply
