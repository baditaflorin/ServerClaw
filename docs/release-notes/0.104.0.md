# Release 0.104.0

- Date: 2026-03-23

- completed ADR 0108 in repository automation by adding the authoritative `config/operators.yaml` roster, the `config/schemas/operators.schema.json` schema, and repo-managed OpenBao operator policy sources under `config/openbao/policies/`
- added `scripts/operator_manager.py` and `scripts/operator_access_inventory.py` to handle onboarding, offboarding, sync, inventory, and quarterly access reviews from the same roster-backed source of truth
- added Windmill wrappers plus Make targets for `operator-onboard`, `operator-offboard`, `sync-operators`, and `quarterly-access-review`, and wired `lv3 operator add`, `lv3 operator remove`, and `lv3 operator inventory` into the platform CLI
- extended the validation gate, repository data-model validation, controller-local secret manifest, ADR metadata, and workstream registry so ADR 0108 is now marked implemented in `0.104.0`
- documented the operational path in `docs/runbooks/operator-onboarding.md` and `docs/runbooks/operator-offboarding.md`

Platform impact:

- no live platform version bump; this release adds repository-side operator lifecycle automation and governance, but live Windmill enablement still depends on deployment-local Tailscale and optional step-ca hook configuration
