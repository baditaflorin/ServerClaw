# Release 0.104.0

- Date: 2026-03-23

## Summary
- implemented ADR 0109 in repository automation by adding the `status_page` service surface, `status.lv3.org` publication contracts, the public status-page runbook, and the matching health-probe and service-catalog entries
- extended the Uptime Kuma wrapper to reconcile the public status page from `config/uptime-kuma/status-page.json`, inspect maintenances, and support ADR 0080 maintenance-window publication into the public page
- added a repo-managed Uptime Robot wrapper plus canonical external-monitor configuration, controller-local secret inventory, and governed workflow or command metadata for independent public monitoring and Mattermost alert-contact reconciliation
- taught the shared NGINX edge publication path to proxy only the root request to an alternate backend path so a dedicated hostname can serve the Uptime Kuma status-page slug cleanly
- updated ADR 0109, its workstream record, and generated status surfaces to mark the repo implementation complete in `0.104.0`
- implemented ADR 0110 in repository automation with machine-readable version semantics, breaking-change criteria, and upgrade policy stored in `config/version-semantics.json`
- added `scripts/release_manager.py` and `scripts/generate_release_notes.py`, wired `lv3 release` into the platform CLI, and documented the release workflow in `docs/runbooks/platform-release-management.md`
- extended the generated ops portal with ADR 0110 release-readiness reporting and added repository validation coverage for the new release-policy schema surfaces
- updated ADR 0110, its workstream, and the release process documentation to mark the repository implementation complete in `0.104.0`

## Platform Impact
- no live platform version bump; this release adds repo-managed status-page, monitoring, documentation automation, release semantics, notes generation, upgrade guidance, and readiness reporting only
