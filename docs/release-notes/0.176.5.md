# Release 0.176.5

- Date: 2026-03-27

## Summary

- integrated ADR 0173 workstream surface ownership manifests and branch validation into the numbered mainline release
- implemented ADR 0179 with a machine-readable `config/service-redundancy-catalog.json` that assigns every managed service an honest `R0`/`R1`/`R2` redundancy tier, RTO/RPO target, backup-source set, standby location, failover trigger, and failback method
- added `docs/schema/service-redundancy-catalog.schema.json`, `scripts/service_redundancy.py`, focused tests, and live-apply preflight hooks so deployment now rejects unsupported redundancy claims before Ansible starts, while staying aligned with the service capability catalog and repository data-model validation

## Platform Impact

- repository version advances to 0.176.5; platform version remains 0.130.22 because the ADR 0179 deployment preflight and documentation changes have not yet been live-applied from current `main`

## Upgrade Guide

- operators can inspect one service with `uv run --with pyyaml --with jsonschema python scripts/service_redundancy.py --service <service-id>`
- future production `live-apply-*` runs will now stop early if a service declares a redundancy tier above the current single-host platform limit
