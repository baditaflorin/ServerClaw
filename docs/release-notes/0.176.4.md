# Release 0.176.4

- Date: 2026-03-27

## Summary

- implemented ADR 0180 with a new standby-capacity validator that checks R2 and higher standby declarations against placement rules, backing capacity, and same-host failure-domain honesty
- extended the service capability and capacity-model schemas so standby metadata is machine-readable and reusable from repo validation, promotion-gate checks, and direct `live-apply-service` execution
- declared the PostgreSQL service as an R2 warm standby backed by `postgres-replica-lv3`, added focused promotion-gate coverage, and added simulated-load tests for standby reservation behavior

## Platform Impact

- repository version advances to 0.176.4; live platform version is unchanged until a production apply from `main`
