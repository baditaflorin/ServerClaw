# Release 0.114.2

- Date: 2026-03-24

## Summary
- completed the ADR 0092 live rollout on top of current `main` by hardening the API gateway runtime on `docker-runtime-lv3`, including Keycloak JWKS access over the shared Docker network, real bearer-token verification during converge, Docker NAT recovery retries, and published-port self-healing
- fixed the guest and edge rollout blockers discovered during live apply: Docker-published traffic is now allowed correctly by the guest nftables forward chain, the Proxmox guest-firewall role now skips absent VMs and supports `vrrp` rules, and the `nginx_edge_publication` role now replaces an existing `lv3-edge.conf` file with the managed symlink
- recorded the ADR 0092 live receipt, updated the ADR and workstream status to `live_applied`, registered the missing `converge-api-gateway` workflow and command contract, and advanced both repository and platform version tracking to `0.114.2`

## Platform Impact
- `https://api.lv3.org/healthz` now serves a valid public HTTPS `200 {"status":"ok"}` response from the live edge certificate that includes `api.lv3.org`
- `https://api.lv3.org/v1/platform/services` now returns `200` with a valid Keycloak-issued bearer token and currently reports `24` catalog services
- the canonical platform state now records ADR 0092 as live-applied at platform version `0.114.2`

## Upgrade Guide
- rerun `ansible-playbook -e proxmox_guest_ssh_connection_mode=proxmox_host_jump playbooks/api-gateway.yml` to reconcile the live gateway runtime from `main`
- rerun `ansible-playbook -e proxmox_guest_ssh_connection_mode=proxmox_host_jump playbooks/public-edge.yml` if the shared edge certificate or `api.lv3.org` publication drifts
