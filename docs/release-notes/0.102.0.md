# Release 0.102.0

- Date: 2026-03-23

- implemented ADR 0104 in repository automation with a canonical service dependency graph at `config/dependency-graph.json`, schema validation, impact analysis, recovery-tier calculation, deployment-order sorting, and Mermaid documentation generated into the architecture section of the docs site
- added operator and integration surfaces for the graph: `lv3 impact <service>`, `scripts/dependency_impact.py`, `scripts/validate_dependency_graph.py`, `scripts/generate_dependency_diagram.py`, and the private FastAPI routes `/v1/platform/dependency-graph` plus `/v1/platform/dependency-graph/{service}/impact`
- extended the generated operations portal with recovery-tier and blast-radius context per service card, added the operator runbook `docs/runbooks/dependency-graph.md`, and updated ADR 0104 plus the workstream registry to mark the repository implementation complete in `0.102.0`
- wired the dependency graph into repository validation through `scripts/validate_repository_data_models.py`, `scripts/validate_repo.sh`, `config/validation-gate.json`, the docs-site generator, and the generated-doc checks so catalog drift or cycle regressions now fail the gate before merge

Platform impact:

- no live platform version bump; this release adds repository-side dependency-graph automation, portal build changes, and private API route additions only
