# Release 0.102.0

- Date: 2026-03-23

- implemented ADR 0103 in repository automation with the canonical `config/data-catalog.json`, the matching JSON schema, and the new `scripts/data_catalog.py` validator wired into the validation gate
- added `scripts/purge_old_receipts.py`, `scripts/decommission_service.py`, and the `lv3.platform.data_retention` role plus `playbooks/data-retention.yml` so repository-managed retention and cleanup policy can be converged and executed from code
- updated monitoring, Mattermost, and NetBox automation to enforce the repository-side retention policy for Loki, Tempo, Grafana annotations, Mattermost data retention settings, and NetBox housekeeping-backed changelog cleanup
- added the dedicated `docs/runbooks/data-retention.md` operational procedure, blocked tracked `.env` files in both preflight and repository-model validation, and updated ADR 0103 plus the workstream registry to mark the repository implementation complete in `0.102.0`

Platform impact:

- no live platform version bump; this release adds repository-side retention controls, decommission cleanup automation, and validation only
