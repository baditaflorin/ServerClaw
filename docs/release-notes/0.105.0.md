# Release 0.105.0

- Date: 2026-03-23

- fixed the collection-scoped mutation audit callback so it resolves `scripts/mutation_audit.py` from the repository root instead of importing a non-existent sibling module path
- fixed NGINX edge certificate SAN parsing so existing domains like `ops.example.com` are recognised correctly and certificate expansion only runs for genuinely missing names
- made `configure-edge-publication` prepare the generated ops portal, changelog portal, and docs portal artifacts before the Ansible copy step so shared edge applies no longer fail when sibling build directories are absent
- completed ADR 0108 in repository automation by adding the authoritative `config/operators.yaml` roster, the `config/schemas/operators.schema.json` schema, and repo-managed OpenBao operator policy sources under `config/openbao/policies/`
- added `scripts/operator_manager.py` and `scripts/operator_access_inventory.py` to handle onboarding, offboarding, sync, inventory, and quarterly access reviews from the same roster-backed source of truth
- added Windmill wrappers plus Make targets for `operator-onboard`, `operator-offboard`, `sync-operators`, and `quarterly-access-review`, and wired `lv3 operator add`, `lv3 operator remove`, and `lv3 operator inventory` into the platform CLI
- extended the validation gate, repository data-model validation, controller-local secret manifest, ADR metadata, and workstream registry so ADR 0108 is now marked implemented in `0.105.0`
- documented the operational path in `docs/runbooks/operator-onboarding.md` and `docs/runbooks/operator-offboarding.md`

Platform impact:

- no platform version bump; this release hardens repository-managed edge publication and mutation auditing, and adds repository-side operator lifecycle automation and governance while keeping the live platform version unchanged
