# Release 0.91.0

- Date: 2026-03-23

- implemented ADR 0090's `lv3` platform CLI at [scripts/lv3_cli.py](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/scripts/lv3_cli.py) with 14 command groups for deploy, validate, status, vm, secret, fixture, scaffold, diff, promote, run, logs, ssh, and open workflows
- added editable installation and refresh targets in [Makefile](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/Makefile) plus [pyproject.toml](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/pyproject.toml) packaging so `lv3` can be installed into the operator environment
- added [docs/runbooks/platform-cli.md](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/docs/runbooks/platform-cli.md) and targeted regression coverage in [tests/test_lv3_cli.py](/Users/live/Documents/GITHUB_PROJECTS/proxmox_florin_server/tests/test_lv3_cli.py)
- updated ADR 0090 and the workstream registry to mark the CLI as implemented in repo version `0.91.0`
- implemented ADR 0084 with a repo-managed `packer/` pipeline for the `lv3-debian-base`, `lv3-docker-host`, `lv3-postgres-host`, and `lv3-ops-base` template catalog
- added `config/vm-template-manifest.json`, `make validate-packer`, and the Windmill helper `config/windmill/scripts/packer-template-rebuild.py` so template validation, rebuild orchestration, and manifest metadata updates run through the same repo-managed path
- switched `roles/proxmox_guests` and the Proxmox inventory model to clone guests from declared `template_key` values instead of creating a Debian cloud template inline during guest convergence
- added regression coverage for the new template manifest helper, guest role behavior, and remote execution environment forwarding used by Packer rebuilds

Platform impact:

- no live platform version change; this release adds the repo-side `lv3` CLI and the repository-managed Packer template pipeline while platform version `0.39.0` remains unchanged until a separate live apply from `main` updates running infrastructure
