# Release 0.157.4

- Date: 2026-03-25

## Summary
- tighten the Docker runtime verification so Windmill converges wait through transient `activating` states instead of failing early during service restarts
- constrain repo-managed Windmill PostgreSQL password generation to URI-safe characters so current `1.662.0` runtimes do not fail on reserved-password bytes in `DATABASE_URL`
- teach the Windmill runtime converge to apply `migrations/0016_idempotency_store.sql` on `postgres-lv3`, verify `platform.idempotency_records`, retry schedule probes, and tolerate the current Windmill script-delete `400 no rows returned` response
- record the first verified ADR 0165 live apply in platform version `0.130.6` and carry forward the already-pending ADR 0154 Windmill convergence notes into a numbered release

## Platform Impact
- repository version advances to `0.157.4`
- platform version advances to `0.130.6` because the production Windmill converge from `main` applied and verified the ADR 0165 idempotency schema on 2026-03-25
