# Release 0.120.1

- Date: 2026-03-24

## Summary
- Fixed the live ADR 0122 Windmill operator admin runtime so the browser app can load the repo-backed operator roster without crashing.
- Corrected the Windmill app route documentation to the workspace-scoped path and hardened the raw app against blocked roster payloads.
- Staged the required worker checkout content onto `docker-runtime-lv3`, mounted it into both worker pools, and changed the operator Windmill wrappers to use `uv run --with pyyaml` plus `PYTHONPATH` instead of the broken `uvx` path.
- Updated the Windmill runtime converge so seeded scripts are deleted and recreated, allowing repo-managed script changes to apply on later converges instead of staying stale forever.

## Platform Impact
- repository version advances to `0.120.1`; live platform version advances to `0.114.5` after the Windmill operator admin hotfix was applied on `docker-runtime-lv3`

## Residual Risk
- The roster and page load are fixed live.
- The inventory panel still returns an error payload because the worker checkout does not have the controller-local Keycloak bootstrap secret files under `.local/`; that is a separate secret-projection design gap, not the original page-load failure.
