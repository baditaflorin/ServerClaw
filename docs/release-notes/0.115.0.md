# Release 0.115.0

- Date: 2026-03-24

- completed ADR 0105 in repository automation by replacing the placeholder ephemeral-only capacity seed with the canonical host capacity model, schema, and `scripts/capacity_report.py`
- added the operator entry points `make capacity-report` and `lv3 capacity`, plus the weekly Windmill wrapper `config/windmill/scripts/weekly-capacity-report.py`
- integrated the capacity model into repository validation and the ADR 0073 promotion gate, while keeping vCPU overcommitment informational and enforcing RAM and disk as blocking capacity limits
- committed repository-side dashboard and alert artifacts for the capacity model in `config/grafana/dashboards/capacity-overview.json` and `config/alertmanager/rules/platform.yml`
- documented the operator workflow in `docs/runbooks/capacity-model.md` and updated ADR 0105 plus the workstream registry to mark the repository implementation as complete in `0.115.0`

## Platform Impact
- no live platform version bump; dashboard, alert, and Windmill application from `main` is still pending

## Upgrade Guide
- pull `main`, then use `make capacity-report` or `lv3 capacity` to inspect the canonical repository-side capacity view before promoting additional workload
