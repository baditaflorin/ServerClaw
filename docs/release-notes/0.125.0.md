# Release 0.125.0

- Date: 2026-03-25

## Summary

- Implemented ADR 0126 as a durable observation-to-action closure loop with explicit `OBSERVED -> TRIAGED -> PROPOSING -> EXECUTING -> VERIFYING -> RESOLVED` transitions.
- Added repo-managed loop state persistence, ledger transition events, and `platform.incident.*` event publication for closure-loop milestones.
- Replaced the Windmill observation-loop placeholder with a real wrapper that turns actionable findings into durable loop runs.
- Added `lv3 loop start|status|approve|close` so operators can inspect, approve, or terminate paused runs without leaving the CLI.

## Platform Impact

- repository version advances to `0.125.0`
- platform version remains unchanged until this release is applied live from `main`

## Residual Risk

- conflict detection is currently service-scoped and repo-local; ADR 0127 resource claims remain a follow-up
- health safety uses the existing `service_health` surface until ADR 0128 lands
- case-library promotion remains manual until ADR 0118 is integrated into the loop
