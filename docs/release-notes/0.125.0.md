# Release 0.125.0

- Date: 2026-03-24

## Summary
- implemented ADR 0128 with a repository-managed composite health index that aggregates service health, SLO, drift, incident, maintenance, and mutation signals into one per-service score
- added the `platform.health` client, the Windmill `refresh_composite` worker, the `health.composite` migration, and disabled seed metadata for the minute refresh schedule
- switched the platform API health surface, the new `lv3 health` command, and the goal compiler gate to the shared composite index, including explicit `HEALTH_UNSAFE` rejection and a recorded `--force-unsafe-health` bypass path
- added focused simulation coverage for composite scoring, the Windmill wrapper, API responses, CLI output, and goal-compiler health gating

## Platform Impact
- repository version advances to 0.125.0; no live platform version bump is recorded because the health schema and Windmill schedule still require apply from `main`
