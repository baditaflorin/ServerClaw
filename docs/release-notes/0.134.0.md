# Release 0.134.0

- Date: 2026-03-25

## Summary
- Implemented ADR 0137 so the shared NGINX edge now renders a universal `robots.txt` policy and `X-Robots-Tag: noindex, nofollow` across the public `example.com` HTTP surface.
- Added explicit `example.com` apex coverage to the shared edge certificate and DNS model so the crawl-policy endpoint is valid over HTTPS instead of falling through to a certificate mismatch.
- Added repository-generated robots meta tags to edge-served HTML surfaces, including the shared landing pages, changelog portal, and docs portal theme override.
- Added role verification and regression tests for `robots.txt`, crawl headers, and robots meta tagging.

## Platform Impact
- repository version advances to `0.134.0`
- platform version remains `0.130.1` until this change is applied live from `main`

## Residual Risk
- crawl controls remain advisory and do not replace authentication, firewall policy, or rate limiting
- the live platform will continue to serve the pre-change crawl behavior until the public-edge and DNS automation is applied from `main`
