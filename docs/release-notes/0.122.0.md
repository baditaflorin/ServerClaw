# Release 0.122.0

- Date: 2026-03-24

## Summary
- Implemented ADR 0141 with a canonical `config/token-policy.yaml` lifecycle contract, a governed `config/token-inventory.yaml`, and controller-side token audit and exposure-response logic in `scripts/token_lifecycle.py`.
- Added Windmill wrappers, receipt directories, and runbooks for weekly token inventory auditing and token exposure response.
- Extended `lv3` with local platform-token storage and expiry status commands plus pre-command warnings when the stored token is expired or inside the 24-hour renewal window.
- Added focused regression coverage for token audit findings, hook-backed revocation and replacement workflows, and the new CLI token lifecycle behavior.

## Platform Impact
- repository version advances to `0.122.0`
- platform version remains `0.114.6` until executable provider hooks are applied live from `main`

## Residual Risk
- the repository-side lifecycle engine is complete, but some live token classes still need provider-specific revocation and rotation hooks configured before the platform can exercise every path without a dry run.
- the broad repository data-model validation still reports a pre-existing unrelated blocker on `main`: `converge-api-gateway` is referenced in the workflow catalog without a matching Make target.
