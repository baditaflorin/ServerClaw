# Release 0.178.12

- Date: 2026-04-05

## Summary
- implements ADR 0359 by replacing 27 near-identical `{service}_postgres` role implementations with a shared `postgres_client` role driven by a canonical `platform_postgres_clients` registry; updates `pg_hba.conf.j2` to generate per-service least-privilege entries; reduces role boilerplate by ~4700 lines
- `platform_postgres_clients` registry in `inventory/group_vars/platform_postgres.yml` is the single source of truth for all service–database relationships, driving pg_hba.conf and role provisioning
- complex roles (windmill, matrix_synapse, superset, temporal, directus, lago) preserve their extra logic while delegating core provisioning to the shared `postgres_client` role

## Platform Impact
- `pg_hba.conf` regeneration required on postgres-lv3 to activate per-service least-privilege entries: `make live-apply-service service=postgres-vm`
- all 27 `{service}_postgres` roles remain fully idempotent; thin wrappers produce identical behaviour to the previous bespoke implementations
