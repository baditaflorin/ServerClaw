# Release 0.114.0

- Date: 2026-03-24

- completed ADR 0097 in repository automation by adding repo-managed Alertmanager and ntfy runtimes, severity-based routing, blackbox-backed probe alerting, and platform runbooks for the first critical alerts
- extended the monitoring stack so Prometheus routes to Alertmanager, keeps the existing SLO blackbox path, and also scrapes generated health-probe targets for readiness and liveness alerting
- added repository validation for alert rules, the `converge-ntfy` workflow contract, and rendered Alertmanager configuration validation via the upstream `amtool` checker
- integrated maintenance windows with Alertmanager silences while preserving the existing Uptime Kuma status-page maintenance synchronization
- updated ADR 0097 and the workstream registry to mark the repository implementation as complete in `0.114.0`

## Platform Impact
- no live platform version bump; repository automation is merged, but the live apply is still pending because the documented SSH path to the platform was not available during this turn
