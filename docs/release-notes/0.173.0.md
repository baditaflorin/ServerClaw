# Release 0.173.0

- Date: 2026-03-26

## Summary

Full live implementation of ADRs 0163–0168: agent discovery infrastructure and handoff protocols. This release makes LLM agents fully self-orienting in this repository with zero tree exploration required.

## What Was Implemented

### ADR 0163 — Repository Structure Index
- Created `.repo-structure.yaml` at repo root
- Machine-readable map of all directories, key files, and cross-cutting concerns
- Includes `for_agents` hints and a 6-step onboarding sequence
- Agents can orient themselves in one read instead of glob exploration

### ADR 0164 — ADR Metadata Index
- Created `docs/adr/.index.yaml` (178 ADRs indexed)
- Created `scripts/generate_adr_index.py` (auto-generates the index)
- Searchable by keyword, concern, implementation status, discovery query
- Regenerated automatically: `uv run --with pyyaml python3 scripts/generate_adr_index.py --write`

### ADR 0165 — Playbook/Role Metadata Standard
- Created `playbooks/.metadata-template.yml` with full header template
- All new playbooks must include `# Purpose:`, `# Use case:`, `# Inputs:`, `# Outputs:`, `# Idempotency:`, `# Dependencies:` fields
- Validated by pre-push gate (ADR 0168 enforcement)

### ADR 0166 — Canonical Configuration Locations Registry
- Created `.config-locations.yaml` at repo root
- Maps every configuration file to its purpose, owner, format, update process
- Covers: infrastructure state, versioning, inventory, automation, service config, agent discovery, project tracking, validation/CI

### ADR 0167 — Agent Handoff Protocol
- Updated `AGENTS.md` with full handoff protocol section
- Onboarding sequence, handoff checklist, commit message format, quick situation table
- All documented under "Agent Onboarding (ADR 0163–0168)" section

### ADR 0168 — Automated Enforcement
- Added `validate_agent_standards` function to `scripts/validate_repo.sh`
- Validates:
  - New playbooks/roles have `# Purpose:` metadata headers
  - Branch appears in `workstreams.yaml` (warning)
  - `docs/adr/.index.yaml` is current when ADRs change (warning)
  - Config registry updated when large config changes detected (warning)
  - Structure index updated when new directories added (warning)
- Available as: `scripts/validate_repo.sh agent-standards`
- Included in `all` validation stage

## Platform Impact

- Repository version: 0.172.1 → 0.173.0
- New discovery files: `.repo-structure.yaml`, `.config-locations.yaml`, `docs/adr/.index.yaml`
- New scripts: `scripts/generate_adr_index.py`
- New template: `playbooks/.metadata-template.yml`
- Updated: `AGENTS.md`, `scripts/validate_repo.sh`, all 6 ADR status fields

## Agent Impact

Any new LLM agent onboarding to this repository can now:
1. Read 6 files in ~2 minutes to be fully oriented
2. Find any ADR by keyword without reading 178 files
3. Find any configuration file without tree search
4. Leave clean handoff context for the next agent
5. Have standards automatically enforced on push

## Upgrade Guide

No migration required. New files are additive.

To regenerate the ADR index after adding new ADRs:
```bash
uv run --with pyyaml python3 scripts/generate_adr_index.py --write
git add docs/adr/.index.yaml
```
