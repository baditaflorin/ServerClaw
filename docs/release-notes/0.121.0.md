# Release 0.121.0

- Date: 2026-03-24

## Summary
- Added ADR 0123 so the health-probe catalog is now the canonical uptime contract for Uptime Kuma, world-state, the platform API, and the interactive ops portal.
- Added `scripts/uptime_contract.py` and switched `config/uptime-kuma/monitors.json` to a generated artifact derived from `config/health-probe-catalog.json`.
- Upgraded the `service_health` collector to use contract-backed HTTP and TCP probes instead of relying on guessed service URLs whenever possible.
- Expanded platform health reporting so the API gateway and interactive ops portal can show live status for active catalog services, not only the smaller set of proxied API services.

## Platform Impact
- repository version advances to `0.121.0`
- platform version remains `0.114.6` until this change is applied live from `main`

## Residual Risk
- command- and systemd-only readiness contracts still require fallback handling in the world-state collector when they cannot be executed from the worker environment.
- the new internal health path is implemented and tested in-repo, but the live platform will not expose the broader service view until this release is deployed from `main`.
