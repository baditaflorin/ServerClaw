# Release 0.110.0

- Date: 2026-03-24

- implemented ADR 0114 in repository automation with a CPU-only incident triage engine that builds a ranked hypothesis set, discriminating checks, and cheapest first action from alert payloads, recent receipts, mutation-audit history, dependency metadata, certificate observations, and optional Loki queries
- added `config/triage-rules.yaml`, `config/triage-auto-check-allowlist.yaml`, the repo-managed Windmill wrappers `run-triage.py` and `calibrate-triage-rules.py`, plus `make triage-alert` and `make triage-calibration` entrypoints for the same contracts
- extended repository validation with triage rule contract checks and added unit coverage for rule evaluation, report emission, and Windmill wrapper behavior in `tests/test_incident_triage.py` and `tests/test_triage_windmill.py`
- documented the operator workflow in `docs/runbooks/incident-triage-engine.md` and updated ADR 0114 plus the workstream registry to mark the repository implementation as complete in `0.110.0`

Platform impact:

- no live platform version bump; this release adds repository-side incident triage automation and rule calibration surfaces that still need scheduling and live apply from `main`
