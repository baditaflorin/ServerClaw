# Release 0.130.1

- Date: 2026-03-24

## Summary
- Finalized ADR 0140 on the public Grafana surface by blocking unauthenticated `https://grafana.example.com/api/health`, keeping dashboard URLs on the login redirect path, and adding `noindex` at the edge.
- Fixed the generated Prometheus latency SLO recording rules to use valid subquery syntax so the monitoring stack can reload the shared SLO rule set.
- Added regression coverage for the latency-rule generator so invalid range-selector syntax does not recur in future releases.

## Platform Impact
- repository version advances to `0.130.1`
- platform version remains `0.114.7` until the ADR 0140 live-apply receipts are recorded from `main`

## Residual Risk
- the shared `make live-apply-service service=grafana env=production` path still depends on the controller-local ntfy Alertmanager password file being present in `.local/ntfy/alertmanager-password.txt` when Alertmanager runtime validation runs in a fresh worktree
