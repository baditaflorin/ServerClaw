# Release 0.129.0

- Date: 2026-03-24

## Summary
- Added ADR 0142 and the repo-managed `public-surface-security-scan` workflow for weekly external validation of the live public HTTP or HTTPS surface.
- Added `scripts/public_surface_scan.py`, `config/public-surface-scan-policy.json`, and the Windmill wrapper `config/windmill/scripts/weekly-security-scan.py`.
- Wired the new scan into the workflow catalog, `platform.security.*` lane metadata, workstream registry, and runbook set.
- Added `receipts/security-scan/` and focused unit tests for target discovery, header checks, auth checks, TLS classification, and event generation.

## Platform Impact
- repository version advances to `0.129.0`
- platform version remains `0.114.6` until this change is applied live from `main`

## Residual Risk
- the first scheduled production run still requires the Windmill worker checkout on the live platform to be updated from `main`
- receipt contents depend on the current public edge state, so expected findings may change as hostnames and headers evolve
