# Release 0.106.0

- Date: 2026-03-24

- implemented ADR 0096 in repository automation by adding the canonical `config/slo-catalog.json`, shared `scripts/slo_tracking.py` helper, and generated Prometheus rule, target, and Grafana dashboard assets for the current platform SLO set
- upgraded the status-page SLO stub introduced by ADR 0109 into the canonical catalog schema so the public `status.example.com` surface participates in the same Prometheus, Grafana, and promotion-gate workflow as the rest of the platform objectives
- extended the monitoring role to install and locally bind `prometheus-blackbox-exporter`, copy the generated SLO assets onto `monitoring`, wire Prometheus rule ingestion and probe discovery, and import the `LV3 SLO Overview` Grafana dashboard
- added `/v1/platform/slos` to the platform context service, rendered an `SLO Status` panel in the static ops portal, and tightened the promotion gate so production promotion is rejected when SLO state cannot be evaluated or any tracked SLO has less than 10% budget remaining
- documented the operator workflow in `docs/runbooks/slo-tracking.md` and updated ADR 0096 plus the workstream registry to mark the repository implementation complete in `0.106.0`

Platform impact:

- no live platform version bump; the repository now ships the SLO stack and monitoring convergence path, but live rollout still requires apply from `main`
