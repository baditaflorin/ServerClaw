# Release 0.94.0

- Date: 2026-03-23

- implemented ADR 0091 in repository automation with a five-source drift suite covering OpenTofu plans, Ansible check-mode changes, runtime Docker image digests, DNS catalog records, and TLS expiry and issuer checks
- added the repo-managed `continuous-drift-detection` Windmill surface, `make drift-report`, `lv3 diff` routing, and committed drift receipts under `receipts/drift-reports/`
- extended the ops portal and the managed Grafana platform overview dashboard with drift status views, and registered the new `platform.drift.*` event lane plus the governing command and workflow metadata
- documented drift-report interpretation, suppression rules, and reconciliation entry points in the new drift-detection runbook
- updated ADR 0091 and the workstream registry to mark the repository implementation complete in repo version `0.94.0`

Platform impact:

- no live platform version bump; this release adds repository-side drift-detection automation and status surfaces that still need live apply from `main`
