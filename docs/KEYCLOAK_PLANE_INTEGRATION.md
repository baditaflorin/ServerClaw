# Keycloak-Plane OIDC Integration (ADR 0353)

## Overview

This integration enables **Single Sign-On (SSO) authentication for Plane task board via Keycloak**, implementing the service integration contract defined in ADR 0353.

## What Was Configured

### 1. **Keycloak Client Setup**
- **Client ID**: `lv3-plane-oidc`
- **Client Type**: OpenID Connect (OIDC)
- **Protocol**: OAuth2 / OpenID Connect
- **Authentication Flow**: Authorization Code Grant

**File**: `collections/ansible_collections/lv3/platform/roles/keycloak_runtime/tasks/plane_client.yml`

**Features**:
- Automatic client creation in Keycloak realm `lv3`
- Secure client secret generation and storage
- Protocol mappers configured for:
  - Email
  - Given name
  - Family name
  - Username
- Redirect URI configuration for both public and internal Plane URLs
- CORS and web origins configuration
- Backchannel logout support

### 2. **Plane Runtime Configuration**
- OIDC environment variables added to plane.env.j2
- Keycloak server URL configuration
- Client credentials management
- Auto signup support
- Session scope configuration

**Files Modified**:
- `collections/ansible_collections/lv3/platform/roles/plane_runtime/defaults/main.yml`
- `collections/ansible_collections/lv3/platform/roles/plane_runtime/templates/plane.env.j2`
- `collections/ansible_collections/lv3/platform/roles/plane_runtime/tasks/main.yml`

### 3. **Integration Contract**
- **File**: `config/integrations/keycloak--plane.yaml`
- **Contract ID**: `keycloak--plane`
- **Integration Type**: OIDC
- **Status**: Active
- **Health Check**: Keycloak OIDC discovery endpoint validation
- **Pre-flight Checks**:
  - Keycloak realm existence
  - Plane OIDC client existence
  - Plane deployment readiness

## Deployment Instructions

### Prerequisites
1. Keycloak instance running and accessible
2. Plane infrastructure deployed on docker-runtime-lv3
3. PostgreSQL available for Plane database
4. Network connectivity between services

### Deployment Steps

**1. Deploy Keycloak (if not already deployed)**
```bash
ansible-playbook playbooks/keycloak.yml -e env=production
```

**2. Deploy Plane with OIDC enabled**
```bash
ansible-playbook playbooks/plane.yml -e env=production
```

The `plane_runtime` role will:
- Create/update the Plane OIDC client in Keycloak
- Generate and secure client secrets
- Render Plane environment with OIDC configuration
- Deploy Plane with SSO enabled

**3. Verify Integration**
```bash
# Check Keycloak client was created
curl -X GET http://10.10.10.20:8093/admin/realms/lv3/clients?clientId=lv3-plane-oidc \
  -H "Authorization: Bearer <ADMIN_TOKEN>"

# Check Plane is responsive
curl -X GET http://10.10.10.20:8093/api/instances/

# Test OIDC discovery endpoint
curl -X GET http://10.10.10.20:8093/realms/lv3/.well-known/openid-configuration
```

## Environment Variables

### In Plane Container

```
# OIDC Server Configuration
OIDC_SERVER_URL=https://keycloak.lv3.org/realms/lv3
OIDC_CLIENT_ID=lv3-plane-oidc
OIDC_CLIENT_SECRET=<auto-generated-secret>
OIDC_DISCOVERY_URL=https://keycloak.lv3.org/realms/lv3/.well-known/openid-configuration
OIDC_CALLBACK_URL=https://tasks.lv3.org/auth/callback/keycloak
OIDC_SCOPES=openid profile email

# Enable OIDC Authentication
ENABLE_OIDC=true
OIDC_ENABLE_AUTO_SIGNUP=true

# Social Auth Configuration (Alternative/Legacy)
SOCIAL_AUTH_KEYCLOAK_OAUTH2_SERVER_URL=https://keycloak.lv3.org
SOCIAL_AUTH_KEYCLOAK_OAUTH2_KEY=lv3-plane-oidc
SOCIAL_AUTH_KEYCLOAK_OAUTH2_SECRET=<client-secret>
SOCIAL_AUTH_KEYCLOAK_OAUTH2_AUTHORIZATION_URL=https://keycloak.lv3.org/protocol/openid-connect/auth
SOCIAL_AUTH_KEYCLOAK_OAUTH2_ACCESS_TOKEN_URL=https://keycloak.lv3.org/protocol/openid-connect/token
SOCIAL_AUTH_KEYCLOAK_OAUTH2_PROFILE_URL=https://keycloak.lv3.org/protocol/openid-connect/userinfo
```

## Authentication Flow

```
1. User visits Plane login page
   ↓
2. User clicks "Login with Keycloak"
   ↓
3. User is redirected to Keycloak login
   ↓
4. User authenticates with Keycloak (or uses existing session)
   ↓
5. Keycloak redirects back to Plane with authorization code
   ↓
6. Plane exchanges code for access token
   ↓
7. Plane fetches user info from Keycloak
   ↓
8. User is logged into Plane (auto-signup if enabled)
   ↓
9. Session established with Plane
```

## Secret Management

### Client Secret Storage
- **Generated in**: Keycloak `lv3-plane-oidc` client
- **Stored in**: OpenBao under `openbao/services/plane/oidc-client-secret`
- **Local Artifact**: `.local/plane/oidc-client-secret.txt`
- **Runtime Injection**: Via Docker Compose with env agent
- **Access Control**: Read-only by plane service account

### Secret Rotation
To rotate the OIDC client secret:

```bash
# 1. Generate new secret in Keycloak
curl -X POST http://10.10.10.20:8093/admin/realms/lv3/clients/<CLIENT_UUID>/client-secret \
  -H "Authorization: Bearer <ADMIN_TOKEN>"

# 2. Update local artifact
echo "<NEW_SECRET>" > .local/plane/oidc-client-secret.txt

# 3. Update OpenBao
vault kv put openbao/services/plane/oidc-client-secret value="<NEW_SECRET>"

# 4. Redeploy Plane
ansible-playbook playbooks/plane.yml -e env=production
```

## Troubleshooting

### "CSRF Verification Failed" Error
**Cause**: Session token mismatch or incorrect redirect URI configuration

**Solution**:
1. Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)
2. Clear browser cookies for Keycloak domain
3. Verify redirect URI matches exactly in Keycloak client settings:
   ```
   https://tasks.lv3.org/auth/callback/keycloak
   https://tasks.lv3.org/auth/oidc/callback
   ```

### Users Cannot Login with Keycloak
**Check**:
1. Keycloak is running and accessible: `curl https://keycloak.lv3.org/.well-known/openid-configuration`
2. Plane has OIDC enabled: `echo $ENABLE_OIDC` in Plane container
3. Client secret is correct: `docker exec plane-api cat /run/secrets/oidc-client-secret`
4. Network connectivity between services

**View Logs**:
```bash
# Keycloak logs
docker compose -f /opt/keycloak/docker-compose.yml logs keycloak

# Plane logs
docker compose -f /opt/plane/docker-compose.yml logs api
```

### "Invalid Client Secret"
**Cause**: Secret mismatch between Keycloak and Plane

**Solution**:
1. Verify secret in Keycloak:
   ```bash
   curl -X GET http://10.10.10.20:8093/admin/realms/lv3/clients/<CLIENT_UUID>/client-secret \
     -H "Authorization: Bearer <ADMIN_TOKEN>"
   ```

2. Update Plane if secret changed:
   ```bash
   ansible-playbook playbooks/plane.yml -e env=production --tags service-plane
   ```
