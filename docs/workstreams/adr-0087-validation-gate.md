# Workstream ADR 0087: Repository Validation Gate

- ADR: [ADR 0087](../adr/0087-repository-validation-gate.md)
- Title: Two-layer push gate — fast local pre-commit + remote parallel check suite — blocking broken code from reaching main
- Status: merged
- Branch: `codex/adr-0087-validation-gate`
- Worktree: `../proxmox_florin_server-validation-gate`
- Owner: codex
- Depends On: `adr-0082-remote-build-gateway`, `adr-0083-docker-check-runner`
- Conflicts With: none
- Shared Surfaces: `.pre-commit-config.yaml`, `Makefile`, `scripts/`, `config/`

## Scope

- update `.pre-commit-config.yaml` to contain only fast (< 5 s) checks: `check-yaml`, `check-json`, `detect-private-key`, `end-of-file-fixer`, `trailing-whitespace`, `gitleaks`
- write `.git/hooks/pre-push` template (managed; generated by `make install-hooks`)
- write `config/validation-gate.json` — authoritative definition of all gate checks, images, commands, severity
- write `scripts/run_gate.py` — reads `config/validation-gate.json`, runs checks in parallel via `parallel_check.py`, emits pass/fail summary
- add `make install-hooks` target (idempotent; safe to re-run)
- add `make pre-push-gate` target (runs the full gate locally without a push)
- add `make gate-status` target (shows which gate checks are enabled and their last recorded result)
- implement bypass logging: `receipts/gate-bypasses/` directory + `post-commit` hook that records `--no-verify` skips
- write Windmill workflow `post-merge-gate` (triggers on merge to `main`, runs the same gate)
- update `make setup` to call `make install-hooks`
- write `docs/runbooks/validation-gate.md`

## Non-Goals

- preventing force-push to `main` (a GitHub/Gitea branch protection rule, not a hook)
- running molecule integration tests in the gate (those run in the Windmill pre-merge workflow, not pre-push)

## Expected Repo Surfaces

- updated `.pre-commit-config.yaml` (fast-only hooks)
- `.githooks/pre-push` (committed template; `make install-hooks` installs it)
- `config/validation-gate.json`
- `scripts/run_gate.py`
- updated `Makefile` (`install-hooks`, `pre-push-gate`, `gate-status`)
- `receipts/gate-bypasses/.gitkeep`

- `docs/runbooks/validation-gate.md`

## Expected Live Surfaces

- `make install-hooks` runs successfully and installs `.git/hooks/pre-push`
- `make pre-push-gate` completes in < 20 s with all checks passing on `main`
- Windmill `post-merge-gate` workflow exists and is enabled on `main`

## Verification

- introduce a deliberate `ansible-lint` violation on a test branch; verify `git push` is blocked with a clear error message
- introduce a deliberate `gitleaks` match; verify the local pre-commit hook catches it before the push even starts
- bypass with `git push --no-verify`; verify `receipts/gate-bypasses/` contains a bypass receipt
- remove `build-lv3` from Tailscale (simulated disconnect); verify fallback to local Docker execution with visible warning

## Merge Criteria

- `make pre-push-gate` passes cleanly on `main`
- the gate blocks at least one known-bad input (tested with synthetic violations)
- `config/validation-gate.json` covers all 8 checks documented in the ADR

## Outcome

- repository implementation is complete on `main` in repo release `0.93.0`
- the hook install path, remote-first pre-push gate, explicit bypass receipts, and Windmill post-merge workflow all now share the same `config/validation-gate.json` manifest
- no live platform version change is claimed; this workstream is repository-only
