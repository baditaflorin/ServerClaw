## Scope

- add repo-managed `n8n_postgres` and `n8n_runtime` roles plus `playbooks/n8n.yml`
- pin the upstream n8n runtime image in `config/image-catalog.json`
- add the `n8n` service to topology, health, workflow, command, dependency, secret, image, and completeness catalogs
- publish `n8n.lv3.org` through the shared edge with editor auth and public webhook prefixes
- document runtime bootstrap, verification, and workflow-management boundaries in `docs/runbooks/configure-n8n.md`

## Non-Goals

- automatic workflow import during converge
- native n8n SSO rollout inside the application itself
- automatic bootstrap of a broad-scope n8n public API key

## Expected Live Surfaces

- `docker-runtime-lv3` serves `http://127.0.0.1:5678/healthz`
- `n8n.lv3.org` serves the editor behind the shared oauth2-proxy and Keycloak boundary
- `https://n8n.lv3.org/webhook/...` remains reachable without edge auth
- the repo-managed owner account can sign in locally after converge

## Merge Criteria

- the n8n runtime converges repeatably with PostgreSQL-backed persistence
- the edge route protects the editor while leaving the documented webhook prefixes reachable
- generated secret handling mirrors the repo-managed bootstrap contract for the runtime
- live verification proves owner bootstrap, edge health, and local readiness

## Outcome

- repository implementation is complete and staged for `main` in repo release `0.148.0`
- first live apply from `main` remains the next step for this workstream

## Live Apply Note

- The integrated `0.148.0` mainline was pushed on 2026-03-25 and `make converge-n8n` was attempted from that integrated state.
- The first live apply is still blocked because Hetzner DNS writes for `n8n.lv3.org` hit the provider brownout window on 2026-03-25 between `11:00` and `13:00` UTC, and the Proxmox host was simultaneously unreachable from the controller on both `ops@100.118.189.95:22` and `ops@65.108.75.123:22` while `https://proxmox.lv3.org:8006/api2/json` timed out.

## Notes For The Next Assistant

- Keep workflow import explicit. Upstream CLI import can deactivate workflows, so repo converge should not quietly reapply workflow JSON files.
- The mixed auth boundary is intentional: `n8n.lv3.org` is edge-protected for humans, but selected webhook and health paths stay unauthenticated for machine callers.
